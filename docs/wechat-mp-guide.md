# 微信公众号推文获取 — 使用指南

本应用支持两种方式获取微信公众号推文，可在 **设置 → 微信 → 获取方式** 中切换。

---

## 方式一：微信读书（默认推荐）

通过微信读书 Web API 间接获取公众号文章。它的优点是门槛低，不需要你额外注册公众号账号；缺点是依赖微信读书的公众号覆盖和书架同步。

### 适合谁

- 只想把常看的公众号接入应用，不想为了抓取文章再注册一个公众号
- 本来就会用微信读书阅读公众号文集
- 接受“先在微信读书里加入书架，再回本应用同步”的流程

### 前置条件

- 已安装并登录微信读书
- 至少有一个目标公众号已经进入微信读书，并被你加入书架

### 如何把公众号放进微信读书

常见做法有两种：

1. **从微信文章跳转过去**
   - 在微信里打开目标公众号的任意文章
   - 尝试使用右上角菜单中的 **「在微信读书中阅读」**
   - 跳转到微信读书后，把该公众号文集/文章 **加入书架**

2. **直接在微信读书里搜索**
   - 打开微信读书 App
   - 在 **书城** 或顶部搜索框中搜索公众号名称
   - 进入对应公众号文集后点击 **加入书架**

> 不同微信/微信读书版本的文案可能略有差异；如果页面名字不完全一致，请抓住这几个关键词：**公众号 / 文集 / 加入书架**。

### 在本应用中的配置步骤

1. 在微信读书中完成公众号关注/加入书架
2. 回到本应用，进入 **设置 → 微信 → 方式一：微信读书**
3. 点击 **扫码登录**，在弹出的页面中完成微信读书登录
4. 登录成功后点击 **从书架同步公众号**
5. 在 **已关注公众号** 列表中确认目标公众号已经出现
6. 回到信息中心刷新微信公众号文章

### 你在本应用里能做什么

- 从微信读书书架同步公众号到本应用
- 查看已同步的公众号列表
- 控制单个公众号是否参与抓取/提醒
- 在信息中心刷新公众号文章

### 常见失败原因与排查

- **扫码成功但同步结果为 0**
  - 通常说明：你在微信里关注了公众号，但还没有把它放进微信读书书架
  - 解决方式：先去微信读书确认该公众号文集已经加入书架，再回来同步

- **只有部分公众号能同步**
  - 原因：微信读书的公众号覆盖并不一定完整
  - 结论：微信里有，不代表微信读书里也有

- **Cookie 已失效 / 校验失败**
  - 解决方式：重新扫码，或在设置页中使用 **刷新 Cookie / 手动配置 Cookie**

### 是否推荐

- **推荐给大多数普通用户**：门槛最低，接起来最快
- 如果你的目标公众号很多，或者微信读书里经常搜不到，再考虑切换到下面的 **公众号平台方式**

### FAQ

#### Q1：我在微信里已经关注公众号了，为什么应用里还是没有？

A：因为这一方式依赖的是 **微信读书书架**，不是微信通讯录里的公众号关注状态。你需要把公众号文章导入微信读书，并在微信读书里加入书架。

#### Q2：为什么推荐列表里没有“一键全部关注”？

A：因为微信读书方式的关注动作发生在微信读书内部，本应用只能负责 **同步书架**，不能直接替你在微信读书里完成关注。

#### Q3：什么时候应该切换到公众号平台方式？

A：当你需要更完整的公众号搜索能力、批量关注能力，或者希望直接在本应用里操作时，可以考虑切换。

## 方式二：公众号平台（功能更强，但门槛更高）

通过 `mp.weixin.qq.com` 的公众平台后台接口直接获取文章。它的优点是搜索更全、可在应用内关注、支持一键批量关注推荐公众号；缺点是你需要先拥有一个能登录公众平台后台的账号。

### 适合谁

- 希望在本应用里直接搜索公众号并关注
- 需要用 **一键全部关注** 批量关注 SSPU 推荐公众号
- 不想依赖微信读书书架
- 能接受先注册并维护一个公众号账号

### 前置条件

- 需要一个可以正常登录 **微信公众平台** 的账号
- 注册时通常需要：
  - 一个未用于公众号注册的邮箱
  - 一个实名认证微信作为管理员微信
  - 按平台页面要求填写的主体信息

### 如何注册微信公众平台账号

根据微信公众平台首页以及微信开放社区现有注册流程说明，典型步骤如下：

1. 在电脑浏览器打开：`https://mp.weixin.qq.com/`
2. 点击右上角 **立即注册**
3. 选择账号类型
4. 填写并激活邮箱
5. 完成信息登记
6. 填写公众号名称、功能介绍、运营地区等账号信息

> 个人使用场景通常会先看 **公众号 / 订阅号** 路线；但账号类型、能力和限制请始终以注册页当时显示的官方说明为准。

> 如果扫码登录时页面提示 **“该微信还未注册公众平台账号”**，说明当前微信下没有可登录账号，需要先完成注册，再回本应用扫码。

### 在本应用中的配置步骤

1. 先确保你已经能在普通浏览器正常进入 `mp.weixin.qq.com` 后台
2. 回到本应用，进入 **设置 → 微信 → 方式二：公众号平台**
3. 点击 **扫码登录**
4. 使用管理员微信完成扫码登录
5. 登录成功后，应用会自动提取 Cookie 和 Token，状态显示为 **已认证**
6. 然后你可以：
   - 手动搜索公众号并关注
   - 或者直接对 SSPU 推荐列表使用 **一键全部关注**

### 在本应用里如何关注公众号

#### 手动关注

1. 在 **搜索公众号** 输入框中输入名称
2. 确认搜索结果中的公众号名称/微信号
3. 点击 **关注**
4. 该公众号会进入本地关注列表，后续刷新时会抓取它的文章

#### 批量关注 SSPU 推荐公众号

1. 选择 **方式二：公众号平台**
2. 确保已认证
3. 在 SSPU 推荐公众号卡片中点击 **一键全部关注**
4. 系统会逐个搜索、自动跳过已关注项，并显示进度

### 你在本应用里能做什么

- 搜索任意公众号
- 手动关注指定公众号
- 一键批量关注 SSPU 推荐公众号
- 管理已关注列表
- 单独开关每个公众号是否参与抓取/提醒

### 常见失败原因与排查

- **扫码页提示没有可登录账号**
  - 先去官网完成注册
  - 注册完成后，再回本应用扫码

- **注册后仍然无法登录**
  - 先确认账号信息已经填写完整
  - 先确认你能在普通浏览器里进入后台首页
  - 若浏览器都进不去，就先不要回应用里排查

- **搜索或批量关注中断**
  - 常见原因：会话过期、接口频率限制
  - 解决方式：重新扫码登录，稍等片刻后再试

- **批量关注时第一个结果不正确**
  - 批量流程是按搜索结果自动匹配的
  - 如果遇到重名或误匹配，建议改用手动搜索并确认微信号后再关注

### 是否推荐

- 如果你追求的是 **搜索更全、应用内直接关注、批量关注**，这一方式更强
- 如果你只是想轻量接入少量公众号，通常还是 **微信读书方式更省事**

### FAQ

#### Q1：个人一定要做额外认证才能用吗？

A：本应用需要的是 **你能正常登录公众平台后台并拿到登录态**。是否还需要做后续认证，取决于你的运营需求以及平台当时规则。

#### Q2：为什么这一方式一定要先注册公众号？

A：因为应用调用的是公众平台后台接口，必须先有一个能登录后台的账号作为入口。

#### Q3：为什么它比微信读书方式复杂？

A：因为它换来了更强的能力：**搜索更完整、可应用内关注、支持一键全部关注**。两种方式本来就是“低门槛”和“强能力”的取舍。

## 两种方式对比

| 特性 | 方式一：微信读书 | 方式二：公众号平台 |
|------|-----------------|-------------------|
| 适合人群 | 普通用户、轻量接入 | 想要更强搜索/批量关注能力的用户 |
| 前置条件 | 微信读书账号 + 书架 | 可登录公众平台的公众号账号 |
| 登录方式 | 微信扫码登录微信读书 | 管理员微信扫码登录公众平台 |
| 会话有效期 | 相对更长 | 相对更短 |
| 文章来源 | 微信读书里可见的公众号 | 公众平台可搜索到的公众号 |
| 应用内直接关注 | 不支持 | 支持 |
| 一键全部关注 SSPU 推荐号 | 不支持 | 支持 |
| 推荐场景 | 快速、省心 | 功能更强、控制更多 |

---

## 共享功能

无论选择哪种方式，以下能力都可用：

- **微信渠道开关**：控制微信来源消息是否参与抓取/提醒
- **单公众号通知开关**：控制单个公众号是否参与抓取/提醒
- **信息中心统一展示**：抓取到的推文都会进入同一套消息列表
- **两套配置相互独立**：切换方式不会覆盖另一方式的认证信息和关注列表

## 总 FAQ

### Q：切换获取方式后，原来的关注列表会丢吗？

不会。两种方式的关注列表是独立存储的，切回来仍然保留。

### Q：可以同时启用两种方式吗？

不可以。同一时间只能选一种抓取方式；但两边的配置都会保留。

### Q：信息中心的微信消息为什么有的显示到分钟，有的只有 `00:00`？

- 微信文章会尽量使用实际发布时间
- 没有精确发布时间的官网类消息，会按规则补齐：
  - 当天消息：记录抓取时刻
  - 非当天消息：记为该日 `00:00`

### Q：如果我只是想“尽量少折腾”，应该选哪个？

优先选 **方式一：微信读书**。

### Q：如果我最在意“覆盖更多公众号”和“批量关注”，应该选哪个？

优先选 **方式二：公众号平台**。
