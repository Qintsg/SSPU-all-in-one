# 微信公众号推文获取 — 使用指南

---

## 前置条件

- 需要一个可以正常登录 `mp.weixin.qq.com` 后台的公众号账号
- 注册时通常需要：
  - 一个未用于公众号注册的邮箱
  - 一个实名认证微信作为管理员微信
  - 按平台页面要求填写的主体信息

> 个人使用场景通常会先看 **公众号 / 订阅号** 路线；账号类型、能力和限制以微信公众平台注册页当前显示为准。

---

## 在本应用中的配置步骤

1. 先确认你可以在普通浏览器正常进入 `mp.weixin.qq.com` 后台。
2. 回到本应用，进入 **设置 → 微信推文**。
3. 点击 **扫码登录**，使用管理员微信完成登录。
4. 登录成功后，应用会自动提取 Cookie 和 Token，并同步更新 `~/.sspu-all-in-one/wxmp_config.toml`。
5. 在 **公众号平台认证** 中点击 **校验有效性**，确认认证仍可访问平台接口。
6. 在 **SSPU 微信矩阵** 中点击未关注项的 **关注** 按钮，或使用 **一键全部关注** 批量关注推荐公众号。
7. 回到信息中心刷新微信公众号文章。

---

## 配置文件与缓存位置

所有用户级配置、认证信息、文章缓存和 WebView2 运行态统一放在：

```text
~/.sspu-all-in-one/
```

关键文件包括：

- `app_state.json`：应用设置、消息缓存、已读状态、关注列表等统一状态文件
- `wxmp_config.toml`：公众号平台认证与高级抓取配置
- `webview2/`：Windows WebView2 用户数据目录

`wxmp_config.toml` 可编辑字段：

- `cookie`：公众号平台 Cookie；扫码登录成功后自动更新
- `token`：公众号平台 Token；扫码登录成功后自动更新
- `app_id`：可选 AppID，供公众号平台接口变体使用
- `user_agent`：公众号平台请求 User-Agent
- `per_request_article_count`：单次文章列表请求条数
- `request_delay_ms`：公众号平台请求间隔，单位毫秒

保存配置文件后，回到设置页点击 **重新加载配置** 即可生效。你也可以使用 **打开配置文件目录** 快速定位目录，或点击 **使用 Visual Studio Code 打开配置文件** 直接在 VS Code 中编辑。点击 **清除认证** 会同步清除状态文件和 `wxmp_config.toml` 中的 Cookie / Token。

---

## 刷新与有效性校验

- 手动刷新微信公众号文章前，信息中心会先校验公众号平台认证有效性。
- 自动刷新微信公众号文章前，后台刷新服务也会先做有效性校验。
- 校验失败时不会继续请求文章接口，避免用失效 Cookie / Token 触发连续失败。
- 若平台提示会话过期或频率限制，请重新扫码登录，或稍后再试。

---

## SSPU 微信矩阵

SSPU 微信矩阵展示应用内推荐的学校相关公众号：

- 未认证时仅展示状态，不允许关注。
- 已认证但未关注的公众号会显示 **关注** 按钮。
- 已关注的公众号显示通知开关，控制该公众号是否参与本应用抓取和提醒。
- **一键全部关注** 会逐个搜索推荐公众号，自动跳过已关注项，并在遇到频率限制时提前结束。

---

## 常见失败原因与排查

- **扫码页提示没有可登录账号**
  - 先去微信公众平台官网完成账号注册和信息登记，再回应用扫码。

- **登录成功后校验失败**
  - 点击 **重新加载配置** 后再试。
  - 若仍失败，重新扫码登录并确认 `wxmp_config.toml` 已更新。

- **刷新时提示认证不可用**
  - Cookie 或 Token 可能已经过期，重新扫码登录。
  - 如果手动编辑过配置文件，确认 `token` 为数字格式且 `cookie` 非空。

- **关注或一键全部关注中断**
  - 常见原因是会话过期或平台频率限制。
  - 稍后再试，或减少连续关注操作。

- **未获取到文章**
  - 确认微信推文渠道已启用。
  - 确认 SSPU 微信矩阵中至少有一个公众号已关注且开关开启。
  - 点击 **校验有效性** 确认当前认证可用。
