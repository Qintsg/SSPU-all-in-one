# 体育部考勤查询规则探索

本文记录 issue #112 实现前对体育部查询系统的只读探索结果，后续维护体育相关查询时以本文为约束。

## 入口与身份

- 体育部查询系统入口为 `https://tygl.sspu.edu.cn/sportscore/`。
- 课外活动考勤明细页为 `https://tygl.sspu.edu.cn/sportscore/stScore.aspx?item=1`。
- 体育部系统与 OA / CAS 独立，不复用 OA 密码。
- 学生身份固定提交 `dlljs=st`，账号字段为 `txtuser`，使用本地保存的学工号。
- 密码字段为 `txtpwd`，使用“体育部查询密码”，该密码与 OA 密码分别保存和读取。

## 登录表单

- 登录页是 ASP.NET WebForms，表单当前提交到 `default.aspx`。
- 必须携带页面当前返回的 `__VIEWSTATE`、`__VIEWSTATEGENERATOR`、`__EVENTVALIDATION` 等隐藏字段。
- 登录按钮是 image input，提交时按浏览器行为带上 `btnok.x` 和 `btnok.y`。
- 登录失败会停留在登录页或返回脚本弹窗提示，应用归类为“体育部账号或密码未通过校验”。

## 编码与解析

- 体育部查询系统页面使用 GBK / GB2312 时代编码，应用通过 `gbk_codec` 解码后再解析 HTML。
- 课外活动考勤总次数按 `早操 + 课外活动 + 次数调整 + 体育长廊` 计算。
- 明细页解析优先读取页面明确标注的分类次数；缺少明确汇总时，按明细记录聚合四类次数。
- 明细记录保留原始表格单元格文本，页面结构变化时仍可在二级页面兜底展示。

## 刷新策略

- 教务中心默认不自动读取体育部查询系统，需点击卡片右下角刷新图标手动读取，卡片右下角同时显示上次刷新时间。
- 设置页“自动刷新设置”可开启体育查询自动刷新并选择刷新间隔；开启后进入教务中心会主动读取一次。
- 每次读取前必须先执行校园网 / VPN 前置检测，检测不可达时不打开体育部登录页、不提交体育部查询密码。

## 只读边界

- 当前能力只允许打开登录页、提交学生身份登录表单、读取课外活动考勤明细页。
- 不执行新增、修改、删除、提交、调整次数等任何写入或状态变更操作。
- UI、日志和文档不得展示体育部查询密码、Cookie 或其它可直接复用的身份值。
